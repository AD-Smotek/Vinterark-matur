# Návrh a realizace jednoduchého zařízení pro IoT

## 1. Internet věcí (IoT)

Internet věcí (IoT – Internet of Things) je koncept, kdy jsou fyzická zařízení připojena k internetu a jsou schopna samostatně komunikovat, sbírat data a reagovat na podněty z okolí bez nutnosti přímého zásahu člověka.

IoT zařízení:
- sbírají data pomocí senzorů
- odesílají data do sítě (cloud, server, aplikace)
- přijímají řídicí příkazy
- mohou spolu vzájemně komunikovat

### Příklady IoT zařízení
- chytrý termostat
- meteostanice
- chytré zásuvky
- bezpečnostní kamery
- senzory teploty, vlhkosti a pohybu

Cílem IoT je automatizace procesů, vzdálené ovládání zařízení a efektivní sběr dat.

## 2. Základní požadavky kladené na IoT zařízení

### 2.1 Hardwarové požadavky
- mikrokontrolér (např. ESP32, Arduino, STM32)
- senzory a akční členy (teplota, relé, LED, motor)
- komunikační modul
  - Wi-Fi
  - Bluetooth
  - LoRa
  - Ethernet
- napájení
  - síťový adaptér
  - baterie (nízká spotřeba)

### 2.2 Softwarové požadavky
- firmware pro řízení zařízení
- schopnost síťové komunikace (HTTP, MQTT)
- možnost aktualizace firmwaru (OTA)
- stabilita a spolehlivost programu

### 2.3 Provozní požadavky
- nízká spotřeba energie
- dlouhodobý provoz bez zásahu uživatele
- odolnost vůči chybám a výpadkům

## 3. IoT zařízení z hlediska bezpečnosti

Bezpečnost IoT zařízení se dělí na provozní bezpečnost (safety) a kybernetickou bezpečnost (security).

### 3.1 Safety (provozní bezpečnost)

Safety zajišťuje, že zařízení:
- neohrozí člověka ani majetek
- správně reaguje na chybové stavy (např. přehřátí)
- obsahuje ochranné prvky

Používané ochrany:
- pojistky
- watchdog
- nouzové vypnutí

Příklad: při poruše se relé automaticky vypne, aby nedošlo k požáru.

---

### 3.2 Security (kybernetická bezpečnost)

Security chrání zařízení proti:
- neoprávněnému přístupu
- útokům z internetu
- zneužití nebo krádeži dat

Používané prostředky:
- šifrování komunikace (TLS, HTTPS)
- autentizace pomocí hesel, certifikátů nebo tokenů
- pravidelné aktualizace firmwaru
- oddělení sítí (firewall, VLAN)

Příklad: IoT zařízení komunikuje se serverem pouze přes šifrované spojení.

## 4. Příklad realizace jednoduchého IoT zařízení

### Zadání
Měření teploty a odesílání naměřených dat do mobilní nebo webové aplikace.

### 4.1 Hardwarová část
- mikrokontrolér ESP32 s integrovaným Wi-Fi
- teplotní senzor DHT22
- napájení přes USB nebo síťový adaptér

### 4.2 Softwarová část
- program vytvořený v Arduino IDE
- čtení hodnot ze senzoru
- připojení k bezdrátové síti
- odesílání dat na server pomocí MQTT nebo HTTP
- zobrazení dat v mobilní nebo webové aplikaci

### 4.3 Komunikace
- zařízení odesílá hodnotu teploty v pravidelném intervalu (např. každých 10 sekund)
- server data ukládá
- uživatel má k datům vzdálený přístup

---

## 5. Závěrečné shrnutí

Internet věcí propojuje fyzická zařízení s internetem a umožňuje jejich vzdálené řízení a monitorování. Základem IoT zařízení je mikrokontrolér, senzory a komunikační rozhraní. Důležitými vlastnostmi jsou nízká spotřeba energie, spolehlivost a bezpečnost, která se dělí na safety a security. Jednoduché IoT zařízení lze realizovat například pomocí mikrokontroléru ESP32.
